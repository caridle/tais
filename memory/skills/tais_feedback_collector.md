# T06: 反馈采集器 SOP

## 核心策略
在每次教学交互中采集结构化反馈数据。

## 采集维度
1. 学生回答正确/错误
2. 学生提出的新问题
3. 学生自主复述概念
4. 误解记录（错误描述 + 正确概念）
5. 学生情绪/挫败感

## 数据格式
- SessionRecord 结构体
- 每轮对话生成一条 Turn
- 概念标签 (concept tag)
- 正确性标记 (correct flag)

## 常见坑点
- ❌ 只采集错误不采集正确 → 正确回答也很重要
- ❌ 采集过于频繁 → 每 3-5 轮采集一次即可
- ❌ 数据不完整 → 确保 session_id, concept, timestamp 齐全
